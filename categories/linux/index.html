    Synchronizing Data in Io Operations

    September 27, 2020
    Linux, TIL
    io, linux

    When reading and writing data using a command such as dd, Linux can read data in larger chunks than requested buffering it in memory. The buffer is known as the dirty cache and can be controlled using the dirty_* parameters defined in the Linux sysctl vm documentation. Dependent on the dirty_* configuration, the amount of dirty cache data can grow quite large.
